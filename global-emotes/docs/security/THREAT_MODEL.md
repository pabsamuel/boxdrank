# THREAT MODEL

STRIDE-style pass over the surfaces; each threat maps to an implemented control or a tracked gap. Data-flow: see `docs/architecture/SYSTEM_OVERVIEW.md` diagram.

| Threat                                   | Vector                                    | Control (implemented unless noted)                                                                                                                                                              |
| ---------------------------------------- | ----------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Keyboard trust / typed-text exfiltration | malicious or buggy keyboard code          | no input-capture APIs in keyboard trees (CI grep job); IME/keyboard perform zero network I/O; iOS `RequestsOpenAccess:false`; analytics allowlist rejects content props (tested)                |
| Session theft                            | cookie theft, DB leak                     | httpOnly+Secure+SameSite=Lax cookies; only SHA-256 of session tokens stored; server-side revocation; 30-day expiry                                                                              |
| Magic-link abuse                         | interception, brute force, enumeration    | 15-min single-use tokens (hash-stored); 5/15min per-IP limit; identical response for unknown emails                                                                                             |
| CSRF                                     | cross-site cookie rides                   | SameSite=Lax + JSON-body content-type requirement + strict CORS origin allowlist; OAuth state bound to session via signed cookie                                                                |
| Malicious upload                         | polyglot files, bombs, spoofed MIME       | magic-byte sniffing, pixel-limit bomb guard, size/frame caps, quarantine bucket until processed, metadata strip by re-encode; malware scanning hook = **gap: add ClamAV lambda before Stage 2** |
| Stored XSS                               | names/bios/pack descriptions              | React auto-escaping on web; HTML-escaping in email templates (tested); no dangerouslySetInnerHTML                                                                                               |
| BOLA/IDOR                                | direct object ids                         | `ownedCreator`/`ownedPack` guards on every mutation (403 tested); admin routes role-gated                                                                                                       |
| Invite-code brute force                  | guessing codes                            | 31^12 unambiguous-alphabet code space; 10/hour per-IP redemption limit; per-code max redemptions + revocation                                                                                   |
| Webhook forgery/replay                   | fake Stripe/Twitch calls                  | HMAC verification (both tested incl. tamper + replay-window cases); inbox tables dedupe by external event id                                                                                    |
| Billing entitlement forgery              | client claims plan                        | plans derive only from webhook-synced subscription state; server-side limit assertions (tested)                                                                                                 |
| Storage URL leakage                      | shared signed URLs                        | short TTL signed GETs; originals never public; asset_cache_grants audit table                                                                                                                   |
| Provider token compromise                | DB dump                                   | AES-256-GCM at rest; key outside DB (`TOKEN_ENCRYPTION_KEY`); never in responses/logs (pino redaction); rotation runbook below                                                                  |
| Report flooding                          | spam reports                              | 5/hour per-IP on public reports; moderation queue                                                                                                                                               |
| Creator impersonation                    | fake handles                              | reserved-handle list; provider-verified ownership; report category `impersonation`                                                                                                              |
| SSRF                                     | provider callbacks fetching attacker URLs | adapters call fixed official hosts only; no user-supplied URLs are fetched server-side                                                                                                          |
| Supply chain                             | malicious deps                            | pinned lockfile, pnpm audit + gitleaks in CI, minimal dependency policy (CONTRIBUTING)                                                                                                          |

## Runbooks

- **Key rotation** (`TOKEN_ENCRYPTION_KEY`): add `v2:` envelope prefix support in `packages/auth`, deploy dual-read, re-encrypt rows in batches (worker job), flip write key, remove old key. Sessions: rotate `SESSION_SECRET` with a scheduled all-session revocation notice.
- **Provider-token compromise**: revoke provider-side (adapter `revokeConnection` for all rows), mark connections `revoked`, force re-auth, audit access via `asset_cache_grants` + logs, notify affected users.
- **Incident response**: see `INCIDENT_RESPONSE.md`.
